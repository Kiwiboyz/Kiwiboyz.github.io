About

Whats the project?

The task of this project was to build a web app that allowed users to see the upcomming semester dates , this allows the user to find out important information such as public holidays, upcoming assignments and the current week of the semester

With a tool like this it should help both students and lectures plan out there semeseter more easily without the stress of forgotten dates

Summary

I will do a write up closer to the end of the project so far i belive we are on the right track as a team and feel good with the amount of work I have done

TimeLine

Week 1

This year I took on a new project called Semester tracker this project was to allow the user to visually see the semester dates assignment, due dates and public holidays, We first to get to know our fellow team members, set up a git hub and get our communication channels working. We added everyone to the new Discord server and put up the links to the GitHub.

.

Lesson 2


Next up we needed to get the original files for the current project. We found it on her Github and got to work on getting it running. Once we got it running we found numerous bugs and added them as bugs to the Github

Week 2

At the start of this week we were about to start on rewriting the whole project in PHP but then Elise recommend we look for a project like yii, at first we were hesitant but got to work at looking for php frameworks but got to work searching for a preferred framework. 

.

Lesson 2


By lesson 2 we had decided to ultimately go for the YII framework. The next step was to start learning how to use YII we spent the rest of this lesson doing this, I also got to work on preplanning the data base.

Week 3

Week 3 we organised ourselves into teams at the time there were two teams a frontend and backend I took on the role of backend but also acted as the manager for the project. My first task was to preplant the database. I started looking at Elise’s old project to get an idea of what we needed. We also planned to go and see Adon our other client to discuss user stories . Unfortunately I was unable to attend this meeting but got notes on what happened.   



Lesson 2


We needed to have a quick catch up with our Client but they were busy with other groups for the most of the lesson, We still hadn’t learned how to use the Yii framework so things were going slow, the data base was still in a predesign but without talking to Elise I couldn’t properly get a handle on what she wanted until we had met. I was able to get a rough design on paper done after class.

Week 4

When we got back from holiday we did a quick scrum I had made progress with the YII framework in understanding the backend- frontend power of it and was amazed by it, I had also done some work in designing a database based off of the original site, admittedly I went away for the holiday so I didn’t get as much done as I wanted to. Over this week I worked with Ryan to combine all the plans of the database into one.



Lesson 2


Myself and Ryan were still stuck on the database the main issue being keeping it third form and relational we were also trying to learn how to use the framework that Aidyn had got running, This class was more dedicated to Aidyn running us through the steps of how to start the site locally.

Week 5

This lesson we finally completed the database all we had to do now was create it. I started to prepare to my home server to set up a virtual webhost and set up a subdomain on the DNS servers.

Lesson 2


Ryan had sent me the draft database although I made several adjustments to this and ended up uploading four different files to the database with multiple changes per version. I ended up cutting a whole heap of redundant tables out because we did not need them and simplified the other data,

Week 6

This week I decided I should get a live version of the site running I had to decide whether I should use the polytechnic network or use my own built on my domains, in the end I decided to work on my own server I had already set up the virtual server space and had room to add a subdomain I chose the home server as I had more control over it and could run it how I liked this also did add the responsibility of keeping the servers up but this was easy enough.

Lesson 2


After a week of issues regarding DNS issues database permission issues and virtual server connection problems I was able to get  the site live, this was the first time the new site was live although I had to make more improvements regarding the database to stabilise some random data values

,

Week 7

This week we made massive strides in progress I had a final version of the database and updated the site to use the newest versions of code I also updated out Go button to link directly to the Login screen once that was done I helped James and Aiden work on the Login screens connecctons to the database I was originally going to have to make a new table as it was not working it turns out James was doing it correctly but was just referencing the wrong table once that was fixed we were able to log in and the site would remember old users. Ryan was able to show Jamie how to use YII rather than write the page manually. We also had a major issue with the Live version. The Bugtracking site was down and I didn’t make backups for an easy restore the site was down for around an hour before I found the logs I had a “6” in a command it must of happened when I was updating the site to the newest version. I had to add some more tables for Jamie's ticket system although he was just back from his sickness so he hadn’t made much progress we were ready to help. Ryan had now created all the data inputs using the Yii framework so we could now add People, semesters, Class’s, and assignments

Lesson 2


Unfortunately, I was unable to attend this day as I was called to an urgent meeting although I  was working on generating dummy data for a more realistic simulation of the site being live while also working with Ryan to set up automated testing. James was also working on testing all of our input fields to see if it could be broken easily. Jamie was adding preselected options for the leave tickets system rather than having to add it manually.

Week 8

This lesson I helped Aidyn and James work on the login system for students and lectures I had to try and explain that the permissions row on the person table was used to define whether it is a student or a lecture trying to login we attempted to use the built in Gii but it didn’t work..

Lesson 2


In this lesson we discussed changing the way person is made and the possibility of adding an extra table for authentication but after going through the YouTube tutorial again Aidyn noticed that we were still pointing towards a user table in a config rather than the persons table. I also did a lot of set up on my portfolio,

Week 9

This week I was pretty busy with other assignments I was staying at polytechnic from around 9:00 am to 10:00 pm each night I was able to fix the lockout that I had accidentally caused by attempting the Server password incorrectly too many times. I checked up on the rest of the team and they said they were at work on the API.

Lesson 2


I have to admit not much happened in class this lesson we mostly just worked on our portfolios and were fixing the odd bugs in the data base I had to make a quick fix to the Database import as it would throw an error ever time we went to set it up on a new machine this was due to an “Add” statement in the wrong position,

Week 10

Aidyn had a meeting with Adon on behalf of the group to show him the new API he had been working on, he asked that we create some way to selectively pull data from the data base and create CSV import options I took a look at the CSV file and noticed it had some data we currently could not handle and got to work and reworking the backend.

Lesson 2


Things had become complicated, it being so close to the end of semester work and other assignments began to build up, I wanted to get the backend database reworked but sadly didn’t have enough time for it. I was able to come up with a few options on how we could rework it but I’d need to some more time to check all the possibility’s .

Week 11

After I met with the group I decided to go with the in my mind best solution to this issue I didn’t want to affect the rest of the teams work. Since most of the site is auto generated by Gii from the database there was really no way that I could make the required adjustments to it without breaking anything. I communicated this in advance with the team to give them a heads up. I also made some major changes around the course we now take the day of the week in to account when we want to make a new course. This will show up on the semester dates page.

Lesson 2


I was working with other team members this lesson guiding them through the new database design and what has been changed what will need to be added and what is already linked. Near the end of this class we just got together and started work on the technical report I took over the formation of it and wrote in how to set up a live server. We also added a list of all the technologies used along with the version.

Week 12

We are pretty much finished now I asked the members present to close any remaining issues and remined the team to fill out the personal evaluations, I sent out the individual stats to each team member this contained the bugs they managed to resolve and an overall view of how well we did over the course of this project. I also updated the Live version to the most recent working local version. My main concern was keeping the site in a stable yet recent version. This meant no obvious bugs or errors. I quickly tested the site and found nothing obvious..
